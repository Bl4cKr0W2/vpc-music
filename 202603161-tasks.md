## 0. Critical Findings (Pre-Existing Bugs / Drift)

> Issues discovered during the audit that must be addressed before or alongside new feature work.

### 0.1 Security — Unauthenticated Public Endpoints

> These routes have **no `auth` middleware at all**. Anyone with a UUID can access them without logging in.

- [x] **`GET /api/songs/:id`** — full song content exposed publicly; add `auth` middleware
- [x] **`GET /api/songs/:id/export/chordpro`** — unauthenticated export; add `auth` middleware
- [x] **`GET /api/songs/:id/export/onsong`** — unauthenticated export; add `auth` middleware
- [x] **`GET /api/songs/:id/export/pdf`** — unauthenticated export; add `auth` middleware
- [x] **`GET /api/setlists/:id`** — full setlist with all songs exposed publicly; add `auth` middleware
- [x] **`GET /api/events/:id`** — event details exposed publicly; add `auth` middleware
- [x] **Write regression tests** — confirmed previously public song/setlist/event detail routes and org routes return 401 without auth token

### 0.2 Security — Observer Can Mutate Everything

> `requireOrgRole()` is only used on **2 routes** (admin router + default-variation PATCH). **27 write endpoints** are unprotected — observers can create, edit, and delete songs, setlists, events, variations, sticky notes, share tokens, and usage records.

- [x] **Add `requireOrgRole("admin", "musician")` to all song write routes** — POST, PUT, DELETE `/api/songs`, import endpoints
- [x] **Add `requireOrgRole("admin", "musician")` to all variation write routes** — POST, PUT, DELETE `/api/songs/:id/variations`
- [x] **Add `requireOrgRole("admin", "musician")` to all setlist write routes** — POST, PUT, DELETE `/api/setlists`, songs, complete, reopen
- [x] **Add `requireOrgRole("admin", "musician")` to all event write routes** — POST, PUT, DELETE `/api/events`
- [x] **Add `requireOrgRole("admin", "musician")` to share management routes** — POST, DELETE, PATCH `/api/songs/:id/share(s)`
- [x] **Add `requireOrgRole("admin", "musician")` to sticky note write routes** — POST, PUT, DELETE
- [x] **Add `requireOrgRole("admin", "musician")` to usage write routes** — POST, DELETE
- [x] **Write role-matrix integration tests** — protected write routes now have owner/admin/musician/observer coverage
